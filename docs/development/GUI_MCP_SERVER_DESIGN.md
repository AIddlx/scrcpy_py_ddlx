## 存在的问题

### 复杂度问题
1. **线程架构复杂**：主线程 (Qt) + MCP 线程 + Scrcpy 客户端线程
2. **状态同步困难**：跨线程的 UI 状态更新
3. **错误处理繁琐**：各种边界情况需要处理

### 对比 scrcpy-companion
| 功能 | GUI 方案 | Companion 方案 |
|------|----------|----------------|
| 配置管理 | 客户端 JSON | 服务端存储 |
| 连接控制 | 客户端发起 | 服务端推送 |
| 状态同步 | 轮询/回调 | 事件驱动 |
| UI 复杂度 | 高（多面板） | 低（单一界面） |

## 线程架构设计
```
┌─────────────────────────────────────────────────────────────┐
│                      主线程 (Qt GUI)                        │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              Qt 事件循环 (app.exec())                 │   │
│  │  - GUI 交互                                          │   │
│  │  - QTimer 定时器                                     │   │
│  │  - 预览窗口渲染 (OpenGL)                             │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
          │                    │
          ▼                    ▼
┌─────────────────┐  ┌─────────────────────────────────────┐
│ MCP HTTP 线程   │  │        Scrcpy 客户端线程             │
│ (独立 Thread)   │  │                                     │
│                 │  │  - 视频解码线程 (daemon)             │
│ Starlette       │  │  - 音频解码线程 (daemon)             │
│ uvicorn         │  │  - 控制通道线程 (daemon)             │
│ HTTP:3359       │  │                                     │
└─────────────────┘  └─────────────────────────────────────┘
```

## 未来改进方向

### 方案 A：简化 GUI
- 移除 MCP 集成，只做连接管理
- 使用单一窗口，不用 Tab
- 配置使用系统设置而非自定义管理

### 方案 B：Web GUI
- 使用 Flask/FastAPI 提供 Web 界面
- 单一服务端，浏览器访问
- 状态管理更简单

### 方案 C：专注 Companion
- 手机端 Companion 作为主要控制界面
- 电脑端只提供 MCP 服务
- 减少电脑端复杂度
