## 1. 技术栈

### 1.1 依赖层次

```
┌─────────────────────────────────────────────────────────────┐
│                    scrcpy_http_mcp_server.py                │
│                       (HTTP MCP 服务层)                      │
├─────────────────────────────────────────────────────────────┤
│  Starlette    │  Uvicorn    │  Threading                    │
│  (Web 框架)   │  (ASGI)     │  (线程安全)                   │
├─────────────────────────────────────────────────────────────┤
│                     scrcpy_py_ddlx                          │
│                       (核心业务层)                           │
├─────────────────────────────────────────────────────────────┤
│  PyAV         │  NumPy      │  SoundDevice    │  ADB        │
│  (解码)       │  (图像)     │  (音频播放)     │  (设备通信) │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 技术选型

| 组件 | 技术 | 选型理由 |
|------|------|---------|
| HTTP 框架 | Starlette | 轻量、异步、ASGI 原生 |
| ASGI 服务器 | Uvicorn | 高性能、标准兼容 |
| 线程安全 | threading.Lock | 简单可靠、全局状态保护 |
| MCP 协议 | JSON-RPC 2.0 | 标准协议、易于调试 |
| 视频解码 | PyAV | FFmpeg 绑定、支持多编码 |
| 音频播放 | SoundDevice | 低延迟、跨平台 |

## 5. 连接模式详解

### 5.0 模式对比与安全建议

```
┌─────────────────────────────────────────────────────────────────────────┐
│                          连接模式选择指南                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                          │
│  ┌──────────────────┐     ┌──────────────────┐     ┌────────────────┐  │
│  │   ADB 隧道模式   │     │    网络模式      │     │  ADB WiFi      │  │
│  │  (adb_tunnel)    │     │   (network)      │     │  (5555端口)    │  │
│  ├──────────────────┤     ├──────────────────┤     ├────────────────┤  │
│  │ ✅ 推荐：USB     │     │ ✅ 推荐：WiFi    │     │ ⚠️ 不推荐      │  │
│  │ ⚠️ 可用：WiFi    │     │    直连          │     │    不安全      │  │
│  └──────────────────┘     └──────────────────┘     └────────────────┘  │
│                                                                          │
└─────────────────────────────────────────────────────────────────────────┘
```

#### 安全建议

| 模式 | 安全性 | 建议 |
|------|--------|------|
| **ADB USB** | ✅ 安全 | **优先使用** - 物理连接，无网络暴露 |
| **ADB WiFi (5555)** | ❌ 不安全 | **不推荐** - 明文传输，易被中间人攻击 |
| **网络模式** | ⚠️ 局域网 | 可用 - 仅在可信局域网使用 |

> **⚠️ 重要安全警告**：
> - ADB WiFi (adb tcpip 5555) 通信是**明文传输**，任何同一网络的人都可以连接
> - 仅在**可信的私有网络**中使用网络模式
> - **生产环境请使用 USB 连接**

### 5.1 ADB 隧道模式 (adb_tunnel)

#### 推荐使用方式

```
┌─────────────────────────────────────────────────────────────┐
│                   ADB USB 连接（推荐）                       │
├─────────────────────────────────────────────────────────────┤
│  1. USB 数据线连接手机                                       │
│  2. 手机开启 USB 调试                                        │
│  3. adb devices 确认连接                                     │
│  4. 调用 connect(connection_mode='adb_tunnel')              │
└─────────────────────────────────────────────────────────────┘
```

#### 内部流程

```
connect(connection_mode='adb_tunnel')
        │
        ├── 1. ADBManager 启动服务端
        │      adb shell CLASSPATH=... app_process ...
        │
        ├── 2. 建立 ADB 隧道
        │      adb forward tcp:27183 localabstract:scrcpy
        │
        ├── 3. Socket 连接 localhost:27183
        │
        ├── 4. 协议握手
        │      • 读取 dummy byte
        │      • 读取设备元信息
        │      • 协商编解码器
        │
        └── 5. 启动数据流
               • 视频解码线程
               • 音频播放线程
               • 控制线程
```

#### 数据流

```
┌─────────┐                      ┌─────────┐
│   PC    │    ADB Forward       │ Android │
│         │◄────────────────────►│         │
│ [Client]│   localhost:27183    │[Server] │
└─────────┘   (单端口隧道)        └─────────┘

特点:
• 所有数据复用同一隧道 (控制 + 视频 + 音频)
• 延迟较高 (ADB 隧道开销)
• 需要 ADB 连接持续保持
• 不支持 stay-alive
```

### 5.2 网络模式 (network) 详解

> 参考: `tests_gui/test_network_direct.py`

#### 核心特点

```
┌─────────────────────────────────────────────────────────────────┐
│                      网络模式核心特点                            │
├─────────────────────────────────────────────────────────────────┤
│  • 纯 TCP/UDP 通信，数据传输不依赖 ADB 隧道                      │
│  • ADB 仅用于部署服务端（push + start）                         │
│  • 服务端用 nohup 启动，USB 拔掉后继续运行                      │
│  • 支持 stay-alive 热连接模式                                   │
│  • 支持 UDP wake 唤醒休眠服务端                                  │
│  • 支持 FEC 前向纠错 (独立控制视频/音频)                         │
│  • 端口分离: TCP 控制 + UDP 视频 + UDP 音频                     │
└─────────────────────────────────────────────────────────────────┘
```

#### 端口分配

| 端口 | 协议 | 用途 |
|------|------|------|
| 27183 | UDP | 设备发现 (DISCOVER) / 唤醒 (WAKE) |
| 27184 | TCP | 控制通道 (触摸、按键等) |
| 27185 | UDP | 视频流 |
| 27186 | UDP | 音频流 |

#### 阶段 1: 部署服务端

```
┌─────────────────────────────────────────────────────────────────┐
│                   网络模式 - 服务端部署                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  前提: 手机通过 USB 连接 (仅部署阶段需要 ADB)                    │
│                                                                  │
│  步骤:                                                          │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │ 1. 检查/推送服务端 APK                                    │  │
│  │    adb push scrcpy-server /data/local/tmp/scrcpy-server.apk│ │
│  └──────────────────────────────────────────────────────────┘  │
│                           │                                     │
│                           ▼                                     │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │ 2. 启动服务端 (nohup 后台运行)                            │  │
│  │                                                           │  │
│  │    adb shell "nohup sh -c '                               │  │
│  │      CLASSPATH=/data/local/tmp/scrcpy-server.apk         │  │
│  │      app_process / com.genymobile.scrcpy.Server 3.3.4    │  │
```

```python
class ScrcpyMCPHandler:
    def __init__(self):
        self._lock = threading.Lock()  # 全局锁

    def call_tool(self, tool_name, arguments):
        with self._lock:  # 所有状态操作都在锁内
            # 1. 检查连接状态
            # 2. 执行工具
            # 3. 返回结果
```

- 防止连接/断开操作冲突

## 7. 设计决策

### 7.1 为什么用 Starlette 而不是 Flask？

| 对比 | Starlette | Flask |
|------|-----------|-------|
| ASGI | ✅ 原生 | ❌ WSGI |
| 异步 | ✅ async/await | ❌ 同步 |
| 性能 | 更高 | 较低 |
| 依赖 | 更少 | 更多 |

### 7.2 为什么用单例 Handler？

```python
handler = ScrcpyMCPHandler()  # 全局单例
```

- 设备连接状态需要跨请求保持
- 避免重复连接/断开
- 线程锁保护共享状态

### 7.3 为什么用 JSON-RPC 而不是 REST？

- MCP 协议标准要求
- 支持批量请求
- 统一的错误格式
- 与 Claude Code 原生兼容

### 7.4 为什么 ADB 隧道和网络模式分离？

| 特性 | ADB 隧道 | 网络直连 |
|------|---------|---------|
| 依赖 | 需要 ADB | 仅需 IP |
| 延迟 | 较高 | 较低 |
| 稳定性 | 依赖 ADB | 更稳定 |
| Stay-alive | ❌ | ✅ |
| 安全性 | USB 安全 | 局域网风险 |

分离允许用户根据场景选择最佳模式。

### 7.5 安全设计决策

#### 为什么不推荐 ADB WiFi (5555)？

```
┌─────────────────────────────────────────────────────────────────┐
│                    ADB WiFi 安全风险                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  问题:                                                          │
│  1. 明文传输 - 所有数据可被监听                                 │
│  2. 无认证 - 任何人都可连接                                     │
│  3. 持久端口 - 5555 端口持续监听                                │
│                                                                  │
│  风险场景:                                                       │
│  • 同一 WiFi 下任何人可 adb connect 你的手机                    │
│  • 中间人可截获屏幕内容和控制命令                                │
│  • 咖啡馆/公共 WiFi 极度危险                                    │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

#### 连接模式安全等级

```
┌──────────────────┬────────────────────────────────────────────────┐
│      模式        │                  安全等级                       │
├──────────────────┼────────────────────────────────────────────────┤
│ ADB USB          │ ✅ 高 - 物理连接，无法远程攻击                 │
├──────────────────┼────────────────────────────────────────────────┤
│ 网络模式         │ ⚠️ 中 - 仅限可信局域网，有暴露风险             │
│ (network)        │      建议仅在家庭/办公网络使用                 │
├──────────────────┼────────────────────────────────────────────────┤
│ ADB WiFi (5555)  │ ❌ 低 - 明文传输，任何人都可连接              │
│                  │      不推荐使用                                │
└──────────────────┴────────────────────────────────────────────────┘
```

#### 推荐的安全实践

```
1. 优先使用 USB 连接
   • 物理连接最安全
   • 不暴露任何网络端口

2. 网络模式仅在可信环境使用
   • 家庭 WiFi 或办公室
   • 避免公共 WiFi

3. 不要使用 adb tcpip 5555
   • 如果必须用，使用后立即 adb usb 关闭
   • 或重启手机

4. 网络模式部署后可拔掉 USB
   • 服务端用 nohup 启动，USB 仅用于部署
   • 部署完成后 USB 可以拔掉
```
